- 使用 `System.Web.Http.HttpServer` 記憶體內伺服器

## 6. API 回應格式驗證

所有 E2E 測試統一驗證回應格式符合 `ApiResponse<T>` 規範：

```json
{
  "success": true,
  "data": { /* T */ },
  "errorMessage": null,
  "traceId": "<guid>",
  "timestamp": "<ISO 8601>"
}
```

驗證項目：
- ✅ `Content-Type: application/json`
- ✅ `success` 欄位存在且為 `true`
- ✅ `data` 欄位包含正確資料型別
- ✅ `traceId` 為非空字串
- ✅ `timestamp` 為有效的 ISO 8601 日期

## 7. 安全架構 E2E 測試說明

> [!NOTE]
> 目前 E2E 測試使用 In-Memory HttpServer，**未啟用 OWIN Middleware Pipeline**。
> OWIN 相關的認證/CORS 中介層在 TestServerFixture 中未配置，
> 因此 E2E 測試聚焦於 Web API Controller → Service 的功能邏輯驗證。
>
> **建議後續**：
> 1. 升級 TestServerFixture 使用 `Microsoft.Owin.Testing.TestServer` 以涵蓋 OWIN Pipeline
> 2. 新增安全相關 E2E 測試案例（測試 401/403 回應）
